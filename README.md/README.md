
# Strategic Plan for Data Quality Management in an Online Retail Company

**Type:** Written Assignment  
**Course:** Data Analytics with Python  
**Author:** Dorothy Opare  
**Date:** 19th July 2025  

---

## Project Overview

This project presents a strategic plan to **enhance data quality management capabilities** within an online retail business. The aim is to address key data quality challenges such as:

- Departmental data silos and communication gaps  
- Inconsistent data storage procedures  
- Absence of data governance and standardized management  
- Poor interdepartmental data sharing  

The plan uses frameworks, techniques, and best practices from both **academic literature** and **real-world examples**, offering practical solutions to improve data integrity, collaboration, and decision-making.

---

## Key Objectives

1. Analyze current data management challenges and their causes  
2. Evaluate the consequences of poor data quality on business operations  
3. Develop actionable strategies using data management frameworks:
   - Data Stewardship  
   - Business Glossary  
   - Metadata Management  
   - Standardized Data Modeling  
   - Master Data Management (MDM)  
   - Centralized Data Architecture  
   - Data Governance Frameworks  
   - Continuous Training and Change Management  
   - Cross-Functional Data Collaboration  
   - Shared Analytical Platforms  
   - Feedback Loops  
4. Provide measurable outcomes aligned with **Capability Level 3 under ISO 8000-61**  

---

## Methodology

The strategic plan was developed by:

- Reviewing **existing internal data practices**  
- Identifying **root causes of data quality issues**  
- Applying **data quality frameworks** from course material  
- Using **academic literature** and **real-world company examples** for validation  

This approach ensures solutions are **both practical and sustainable**, balancing technology, processes, and people.

---

## Highlights

- **Data Silos & Communication Gaps:** Implemented Data Stewardship, Business Glossary, and Metadata Management to improve cross-department collaboration.  
- **Inconsistent Data Storage:** Applied Standardized Data Models, Master Data Management, and Centralized Data Architecture to ensure consistency and reliability.  
- **Lack of Governance:** Introduced Data Governance Frameworks, Continuous Training, and Change Management to create a data-driven culture.  
- **Poor Interdepartmental Sharing:** Established Cross-Functional Teams, Shared Analytical Platforms, and Feedback Loops to enhance collaboration and transparency.  

---

## Outcomes

- Improved **data accuracy, consistency, and reliability**  
- Enhanced **decision-making and operational efficiency**  
- Promoted **collaboration and accountability** across departments  
- Achieved a **measurable benchmark** for data quality maturity (Capability Level 3 under ISO 8000-61)
